# v4.0 决策增强升级记录（2026-06-25）

## 问题

决策路由（decision-router.js）有26条规则 + U/D/A/H场域追踪，但：
1. **决策不被执行** — drDecision 只是 think() 返回值的一个字段，没有任何代码根据决策类型改变行为
2. **输入信号弱** — 传给决策路由的数据大量 undefined/fallback 假值（identityCoherence=0.8或0.4, dissonance=信号数×0.15）
3. **规则不学习** — 26条规则权重永远不变，好规则坏规则一样优先级

## 三个新模块

### decision-executor.js
- **路径**: `src/core/decision-executor.js`（410行，新文件）
- **决策→行为映射**:
  - `pause` → depth=1, routeHint.confidence=0.3
  - `accelerate` → depth+1 (max 5), confidence+0.15
  - `turn` → 切换 routeHint.type (general→analytical→creative→general)
  - `hold` → 冻结，设 hold flag
  - `heal` → depth=1, 尝试调 selfHealing 模块, 设 heal_requested flag
  - `resonate` → depth+2 (max 5), type='resonant', confidence+0.2
  - `transmit` → depth=1, type='transmit'
  - `rest` → depth=0, skipThoughtChain flag（think() 检测到后直接返回）
- **安全**: 所有方法 try/catch 包裹，失败返回未修改上下文

### field-injector.js
- **路径**: `src/core/field-injector.js`（430行，新文件）
- 注入 `_fieldU`, `_fieldD`, `_fieldA`, `_fieldQuality`, `_fieldReliable`, `confidence`
- `extractSignals()` 扫描所有 key 匹配 pattern 列表 + 布尔字段 + 字符串字段

### decision-feedback.js
- **路径**: `src/core/decision-feedback.js`（553行，新文件）
- `recordOutcome(decision, wasCorrect, notes)` — 记录决策效果，自动调权重
  - correct → weight+0.05 (max 1.5), wrong → weight-0.10 (min 0.1)
- `adjustPriorities()` — 准确率>80%提权10，<40%降权10
- `save(filePath)` / `load(filePath)` — JSON 持久化

## 集成到 heartflow.js

构造函数（~line 662）：require + new 三个模块
think()（~line 1672）：
1. field-injector 增强输入信号 → 传给 decisionRouter.evaluate()
2. 获取 drDecision → decisionExecutor.apply() 修改 depth/_routeHint
3. REST 决策 → 检测 skipThoughtChain flag → 跳过 ThoughtChain 直接返回
