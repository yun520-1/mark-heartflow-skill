## 修复内容

### 1. 决策路由数据注入增强（pipeline.js）

pipeline decision阶段从4个字段扩展到9个字段：

| 字段 | 来源 | 计算方式 |
|------|------|----------|
| cognitiveLoad | agentPsych + psychology | 多源汇聚 |
| directionClear | judgment | act→0.8, analyze→0.6, 其他→0.3 |
| quality | judgment | judgment.confidence |
| dissonance | agentPsych | cognitiveDissonance.count×0.2 |
| stability | agentPsych | identityDrift→0.3, 否则0.7 |

| desireDominant | deepCognition | desire.dominantDesire |
| poisonLevel | deepCognition | threePoisons.totalToxicity |

### 2. 决策路由兜底规则（decision-router.js）

当无规则匹配时不再返回 `{ decision: null }`，改为返回 `hold(0.3)`。

```javascript
// 在 evaluate() 的 matches.length === 0 分支
return {
  decision: {
    type: DECISION.HOLD,  // 'hold'
    confidence: 0.3,
    priority: 30,
    rationale: '无匹配规则，等待更多数据',
    ruleId: 'default-hold',
    timestamp: Date.now(),
    source,
    fallback: null,
  },
  matched: false,
  rules: [],
  field: fieldData,
};
```

### 3. 记忆系统回填修复（pipeline.js output阶段）

根因：`hf.memory.store('learned', key, value)` 把'learned'当key用，不是写入learned层。

修复：正确调用 store(key, value, tags)，含core:前缀写CORE层。首次运行时自动写入3条CORE规则。

效果：从 core=0/learned=0/ephemeral=0 → core=20/learned=17/ephemeral=1。

### 4. 英文推理增强（logic-reasoning.js）

- `deductive`的minKeywords从3降到2
- 新增30+英文数学运算关键词（calculate/total/sum/difference/find/solve/how many等）
- regexBonus新增3条英文数学正则
- `_matchKeywords`从 `input.includes(kw)` 改为 `input.toLowerCase().includes(kw.toLowerCase())`

### 5. 置信度区分度（judgment-engine.js）

`_scoreDimension` 新增动态评分：
- feasibility：输入长度>100字+1分，>50字+0.5分
- consequence(empathize)：|sentiment|>0.5时+2分，>0.2时+1分
- risk(analyze)：有谬误检测时risk-1
- alignment(analyze)：无明确意图时-1

效果：不同输入产生不同置信度（0.5-0.61），不再是固定0.6。

## 关键教训

### 端到端测试心虫的正确方式

```javascript
// 入口不是 heartflow.js，是 src/core/heartflow.js
const { HeartFlow } = require("./src/core/heartflow.js");
const hf = new HeartFlow();
await hf.start();

// think() 是 async，必须 await
const r = await hf.think("测试输入");

// decision 数据在 r.cognition.decision，不是 r.decision
// r.decision 是 pipeline output 的 direction 回退（'analyze'等）
// r.cognition.decision 是 decisionRouter 的实际输出
console.log(r.cognition?.decision?.type);  // 'hold' | 'pause' | 'accelerate' | null
console.log(r.cognition?.decision?.confidence);
console.log(r.cognition?.decision?.ruleId);
```

### MCP server 记忆层显示0是正常的

MCP 的 health check 不走 pipeline，所以不触发记忆写入。记忆只在 `think()` 的 pipeline output 阶段写入。这是预期行为，不是 bug。

### decisionRouter 无匹配时返回 null 是设计行为，不是 bug

设计意图是"有数据才决策"。但对于所有输入都无数据的情况，需要兜底规则输出 hold。不要改设计为"默认输出 accelerate"——那会掩盖数据流断裂的问题。
