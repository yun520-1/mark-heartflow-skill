   - `scripts/sync_github.py` 从项目内文件读取升级计划

## 📋 升级策略

### 版本号规则

```
主版本。次版本.修订版本
  ↑      ↑      ↑
  |      |      └─ 补丁修复 (安全补丁、bug 修复)
  |      └──────── 功能更新 (新引擎、新特性)
  └─────────────── 重大变更 (架构重构、API 变更)
```

### 触发规则

| 变更类型 | 版本递增 | 示例 |
|----------|----------|------|
| 安全补丁 | +0.0.1 | 10.7.0 → 10.7.1 |
| Bug 修复 | +0.0.1 | 10.7.0 → 10.7.1 |
| 新引擎 | +0.1.0 | 10.7.0 → 10.8.0 |
| 架构变更 | +1.0.0 | 10.0.0 → 11.0.0 |

### 自动化升级

