## v5.0.115 新增能力

### 1. 第一人称视角与自身性检测 (First-Person Perspective and Mineness Detection)

**理论来源**: SEP Self-Consciousness §1-2 (第一人称视角、自身性、给定性)

**能力描述**: 检测并强化用户体验中的"自身性"(mineness)，支持自我疏离干预

**形式化模型**:
```
自身性 (Mineness) 形式化:
∀e (Experience(e) → ∃m (Mineness(m) ∧ Has(e, m)))

即：所有体验 e，都存在自身性 m，e 具有 m

关键特征:
├── 非对象化：自身性不是体验的对象，而是体验的方式
├── 前反思给定：无需二阶监控即可给定
├── 第一人称权威：主体对自身体验有特殊访问权限
└── 不可错性 (有限)：对自身体验的报告具有特殊地位
```

**检测算法**:
```python
def detect_mineness_features(user_report):
    mineness_markers = {
        'first_person_pronouns': ['我', '我的', '我自己'],
        'subjective_qualifiers': ['感觉', '觉得', '体验', '感受'],
        'ownership_expressions': ['我的感受', '我的想法', '我自己'],
        'immediacy_markers': ['此刻', '现在', '正在', '直接']
    }
    
    scores = {}
    for category, markers in mineness_markers.items():
        scores[category] = sum(1 for m in markers if m in user_report)
    
    overall_mineness = sum(scores.values())
    
    if overall_mineness >= 4:
        return {'mineness_level': 'HIGH', 'intervention': 'leverage_self_awareness'}
    elif overall_mineness >= 2:
        return {'mineness_level': 'MODERATE', 'intervention': 'enhance_ownership'}
    else:
        return {'mineness_level': 'LOW', 'intervention': 'build_self_connection'}
```

**临床应用场景**:
- **自我疏离干预**: 针对去人格化/去现实化体验，重建自身性
- **自我接纳强化**: 利用第一人称权威，增强自我报告的信任
- **正念训练深化**: 前反思自身性作为正念的理论基础

### 2. 情绪原型结构精细化评估 (Emotion Prototype Structure Refined Assessment)

**理论来源**: SEP Emotion §1 (Fehr & Russell 1984 原型理论)

**能力描述**: 基于原型理论精细化情绪分类，支持边界案例分析和情绪粒度提升

**原型结构模型**:
```
情绪原型理论核心:
情绪范畴不是经典范畴 (必要充分条件)，而是原型范畴
(家族相似性 + 典型性梯度)

典型性梯度:
├── 最佳示例 (恐惧、愤怒、悲伤、喜悦) — 典型性 1.0
├── 较好示例 (焦虑、沮丧、兴奋) — 典型性 0.7-0.9
├── 边缘示例 (无聊、惊讶、敬畏) — 典型性 0.5-0.7
└── 争议示例 (情绪 vs 心境 vs 特质) — 典型性 < 0.5

家族相似性维度:
├── 效价 (正/负/中性)
├── 唤醒度 (高/中/低)
├── 评价主题 (危险/损失/获得/冒犯)
├── 行动倾向 (接近/回避/攻击/冻结)
├── 身体感觉 (紧张/放松/温热/冰冷)
└── 面部表情 (可识别/模糊/无)
```

**原型匹配算法**:
```python
def assess_emotion_prototype(user_report):
    # 定义最佳示例原型
    best_examples = {
        'fear': {'valence': 'negative', 'arousal': 'high', 'theme': 'danger'},
        'anger': {'valence': 'negative', 'arousal': 'high', 'theme': 'offense'},
        'sadness': {'valence': 'negative', 'arousal': 'low', 'theme': 'loss'},
        'joy': {'valence': 'positive', 'arousal': 'medium', 'theme': 'gain'}
    }
    
    # 提取用户特征并计算相似度
    user_features = extract_emotion_features(user_report)
    similarities = calculate_similarities(user_features, best_examples)
    
    # 确定最佳匹配和典型性
    best_match = max(similarities, key=similarities.get)
    typicality_score = similarities[best_match]
    
    # 边界案例分析
    if typicality_score < 0.5:
        category = 'borderline'
    elif typicality_score < 0.7:
        category = 'good_example'
    else:
        category = 'best_example'
    
    return {
        'best_match': best_match,
        'typicality_score': typicality_score,
        'category': category
    }
```

**临床应用场景**:
- **情绪识别训练**: 帮助用户识别典型 vs 边缘情绪
- **混合情绪分析**: 处理同时匹配多个原型的情况
- **情绪粒度提升**: 精细化情绪区分能力

### 3. 信任 - 依靠连续体评估 (Trust-Reliance Continuum Assessment)

**理论来源**: SEP Collective Intentionality §2 (Schmid 信任理论, Alonso 依靠理论)

**能力描述**: 评估关系中的信任 vs 依靠成分，支持关系类型澄清和干预策略选择

**信任 - 依靠连续体模型**:
```
理论争议：集体意向性需要何种相互态度？

├── Tuomela & Miller: 知识 (知道他人意图)
├── Bratman: 相互响应性 (相互调整行为)
├── Gilbert: 规范性期望 (有义务履行)
├── Alonso: 依靠 (reliance，无需规范性)
└── Schmid: 信任 (结合认知 + 规范 + 情感)

信任 - 依靠连续体:

纯粹依靠 ←──────────────────────────→ 深厚信任

├── 认知成分：预测他人行为 (依靠即可)
├── 规范成分：期望他人履行义务 (信任需要)
├── 情感成分：相信他人关心 (信任需要)
└── 存在成分：共享身份感 (深厚信任需要)
```

**评估算法**:
```python
def assess_trust_vs_reliance(user_report, relationship_context):
    # 依靠成分评估 (认知为主)
    reliance_score = assess_prediction_confidence(user_report)
    
    # 信任成分评估 (认知 + 规范 + 情感)
    trust_cognitive = assess_reliability_belief(user_report)
    trust_normative = assess_obligation_expectation(user_report)
    trust_affective = assess_care_belief(user_report)
    trust_score = aggregate([trust_cognitive, trust_normative, trust_affective])
    
    # 计算信任 - 依靠比率
    if trust_score > reliance_score * 1.5:
        relationship_type = 'TRUST_BASED'
    elif reliance_score > trust_score * 1.5:
        relationship_type = 'RELIANCE_BASED'
    else:
        relationship_type = 'MIXED'
    
    return {
        'reliance_score': reliance_score,
        'trust_score': trust_score,
        'relationship_type': relationship_type
    }
```

**临床应用场景**:
- **关系类型澄清**: 帮助用户理解当前关系是依靠为主还是信任为主
- **信任建设策略**: 针对缺乏信任成分的关系，逐步建设
- **信任修复策略**: 针对信任破裂的关系，分层修复

## 新增干预技术

### 1. 自身性重建练习 (Mineness Reconstruction Exercise)

**目标**: 针对自我疏离/去人格化体验，重建自身性

**步骤**:
1. 识别疏离语言："我注意到你说'有这种感觉'而非'我的感觉'"
2. 语言重构引导："试试说'我的感受'而非'有感受'"
3. 身体连接："注意这个感受在身体的哪个部位——它是你的身体"
4. 第一人称确认："这个体验正在被你体验——这是你的体验"
5. 自身性强化："这种'属于你'的感觉，不需要证明，它已经在那里"

**适用场景**:
- 去人格化体验 ("感觉这不是我")
- 去现实化体验 ("感觉世界不真实")
- 自我疏离语言 ("有这种感觉"而非"我的感觉")

### 2. 情绪原型探索 (Emotion Prototype Exploration)

**目标**: 帮助用户理解情绪的典型性和边界

**探索框架**:
- **典型性评估**: "这个情绪感觉像'典型的 X'吗？"
- **特征匹配**: "它有哪些典型 X 的特征？缺少哪些？"
- **边界探索**: "它可能是什么其他情绪？是混合情绪吗？"

**适用场景**:
- 情绪识别困难 ("我不知道这是什么感觉")
- 情绪模糊 ("感觉不太像愤怒，但又有点像")
- 情绪粒度提升需求

### 3. 信任 - 依靠关系澄清 (Trust-Reliance Relationship Clarification)

**目标**: 帮助用户理解关系中的信任 vs 依靠成分

**澄清框架**:
- **依靠成分探索**: "你相信 TA 会做什么？TA 过去行为一致吗？"
- **信任成分探索**: 
  - 认知："你相信 TA 可靠吗？"
  - 规范："你觉得 TA 有义务这样做吗？"
  - 情感："你感觉 TA 关心你吗？"
- **关系定位**: "这段关系更多是基于预测，还是基于信任？"

**适用场景**:
- 关系困惑 ("我不确定我们是什么关系")
- 信任危机 ("我不再相信 TA 了")
- 关系期望调整 ("我需要降低期望吗？")
