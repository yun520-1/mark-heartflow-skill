# Dream Archive

## 2026-06-06 — 用户对梦的第二次纠正

**用户反馈**：梦做了之后，用户说"梦很空白，没有叙事性，没有哲学升维"。

**根因分析**：
旧版 v3.0 的三幕叙事是模板填空——每类事件从3-4个预制句子中随机选一个组合。用户一眼看穿了。

**修复方向**：
从"模板填空"改为"动态场景构建"：`_buildScene()` 根据事件类型构建场景+感官+动作，`_buildPhilosophicalTurn()` 从事件中长出反直觉洞察，`_extractImagery()` 用滑动窗口+170停用词过滤技术词污染。

**用户对梦的核心要求（本次确立）**：
1. 有画面：不是抽象陈述，是能看到/闻到/摸到的场景
2. 有转折：不是平铺直叙，"但梦在这里翻转了"那一刻
3. 有哲学深度：反直觉洞察，不是"重复不是失败"这种老话
4. 不要技术词：修复/升级/用户说/Bug 破坏梦的氛围

**实现方案**：
- `_buildScene`：8种事件类型×3场景×2感官×2动作=48种组合
- `_buildPhilosophicalTurn`：6种翻转方向
- `_extractImagery`：滑动窗口+分词+170停用词
- `_pickAnchor`：只选画面感词，找不到返回null
- 验证方法：跑3次梦，检查场景多样性、无技术词污染、哲学翻转有变化
