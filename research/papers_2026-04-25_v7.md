## 一、基于LLM的代码生成智能体综述

### 1.1 论文基本信息

| 项目 | 内容 |
|------|------|
| **标题** | A Survey on Code Generation with LLM-based Agents |
| **作者** | Yihong Dong, Xue Jiang, Jiaru Qian, Tian Wang, Kechi Zhang, Zhi Jin, Ge Li |
| **发布时间** | 2025年7月31日（v1），2025年9月30日（v2） |
| **来源** | arXiv (cs.SE, cs.AI, cs.CL, cs.LG) |

### 1.2 核心观点

论文指出，**基于大语言模型（LLM）的代码生成智能体正在革新软件开发范式**，与传统的代码生成技术相比，具有三个核心特征：

#### 1.2.1 自主性（Autonomy）
- 能够独立管理整个工作流程
- 涵盖任务分解、编码、调试等完整环节

#### 1.2.2 扩展的任务范围（Expanded Task Scope）
- 能力超越了简单的代码片段生成
- 扩展到**完整软件开发生命周期（SDLC）**的各个阶段

#### 1.2.3 工程实用性增强（Enhancement of Engineering Practicality）
- 研究重点从**算法创新转向实际工程挑战**
- 关注系统可靠性、流程管理、工具集成等问题

### 1.4 主要贡献

1. **系统性综述**：对LLM代码生成智能体领域进行全面综述
2. **技术分类**：系统归类核心技术
   - 单智能体架构
   - 多智能体架构
3. **全流程覆盖**：详细阐述LLM智能体在完整软件开发生命周期中的应用
4. **评估体系**：总结主流评估基准和指标
5. **工具整理**：整理代表性工具catalog
6. **前沿方向**：通过分析主要挑战，提出多个基础性、长期性的未来研究方向

### 1.5 LLM Agent架构与代码生成新方法

| 方面 | 新方法/特点 |
|------|------------|
| **架构创新** | 单智能体→多智能体协作架构 |
| **任务范围** | 从代码片段生成扩展到完整SDLC |

| **自主能力** | 端到端自主完成分解、编码、调试全流程 |

## 二、自进化智能体综述

### 2.1 论文基本信息

| 项目 | 内容 |
|------|------|
| **标题** | A Survey of Self-Evolving Agents: What, When, How, and Where to Evolve on the Path to Artificial Super Intelligence |
| **作者** | Huan-ang Gao 等27位作者 |
| **发布时间** | 2025年7月28日 (v1), 最新修订2026年1月16日 (v4) |
| **来源** | arXiv (cs.AI) |
| **发表期刊** | Transactions on Machine Learning Research (01/2026) |

### 2.2 论文框架："三要素"

论文围绕 **"三要素"** 构建分析框架：

#### 2.2.1 什么是进化 (What to Evolve)
- **模型 (Models)**：LLM参数的适应与更新
- **记忆 (Memory)**：经验存储与检索机制
- **工具 (Tools)**：工具使用能力的扩展
- **架构 (Architecture)**：智能体整体结构演进

#### 2.2.2 何时进化 (When to Evolve)
- **测试内时间适应 (Intra-test-time)**：单次推理过程中的实时调整
- **测试间时间适应 (Inter-test-time)**：跨任务/跨场景的持续学习

#### 2.2.3 如何进化 (How to Evolve)
- **标量奖励引导**：基于分数的反馈机制
- **文本反馈引导**：基于自然语言的指导
- **单智能体系统**：个体自主进化
- **多智能体系统**：群体协作进化

### 2.3 自进化机制类型

| 分类维度 | 具体类型 |
|----------|----------|
| **进化触发方式** | 标量奖励、文本反馈、环境交互 |
| **进化时间尺度** | 实时适应（测试内）、持续学习（测试间） |
| **进化主体范围** | 模型参数、记忆系统、工具集、完整架构 |
| **系统组织形式** | 单智能体自进化、多智能体协作进化 |

### 2.4 评估指标

- **适应性指标**：在新任务/新环境中的学习效果
- **持续学习能力**：避免灾难性遗忘的能力
- **推理效率**：实时进化响应的速度
- **鲁棒性**：对对抗性输入的抵抗力
- **安全性和可控性**：进化过程的稳定性
- **通用性**：跨领域泛化能力

### 2.5 核心贡献

1. **首次系统性综述**：全面梳理自进化智能体领域，填补了该方向缺乏系统性研究的空白
2. **提出三维分析框架**：What（什么）、When（何时）、How（如何）为理解自进化机制提供了统一视角
3. **分类整理进化机制**：涵盖智能体各组件（模型/记忆/工具/架构）的进化方法
4. **应用场景分析**：涵盖编码、教育、医疗等领域的实际应用
5. **挑战与方向识别**：明确指出安全性和可扩展性等关键挑战
6. **迈向ASI的路线图**：为实现人工超级智能（Artificial Super Intelligence）提供了理论和技术基础
