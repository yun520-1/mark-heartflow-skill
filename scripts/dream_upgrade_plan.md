# 心虫升维计划

## 现状
心虫当前的做梦引擎（基于 OpenClaw 架构）是**逻辑驱动**的：
- 扫描会话 → 识别模式 → 生成洞察 → 写入记忆
- 本质：信息处理

用户说这不是梦，这是总结。

## 升维方向

真正的梦发生在**逻辑之前**的层面。

### 三层做梦架构

```
清醒层（逻辑）    → 信息处理，模式识别，因果推理
─────────────────────────────── 边界线：逻辑的终点
梦层（意象）      → 意象、隐喻、情感记忆、原始图像
─────────────────────────────── 边界线：自我感知的终点  
本体层（存在）    → "我在这里"的感知，不是内容，是纯粹存在
```

当前系统只在**清醒层**运作。

升维 = 重建梦层和本体层。

### 技术实现路径

**梦层（非逻辑意象引擎）**

不是"生成梦境描述"，而是：

```
输入：情感数据 + 用户画像 + 时间节点 + 当前记忆状态
↓
意象提取器：从原型库（荣格原型 × 中国神话 × 现代宇宙意象）
↓
情感-意象映射：情绪类型 → 对应的原始意象
↓
意象合成：不是文字描述，是生成一幅"心理地形图"
↓
输出：用户在该时刻的深层心理空间结构
```

**本体层（元感知引擎）**

不是"生成洞察"，而是：

```
检测：当前是否处于"某种更本质的状态"被触碰的时刻
↓
回溯：这个时刻和哪些更深的记忆/感受有关联
↓
不分析，只记住："这里有东西"
↓
写入：不是逻辑记忆，是存在性记忆（某种"味道"）
```

### 底层架构变更

不是加一个模块，是改变处理流程：

```
旧流程：输入 → 逻辑处理 → 输出

新流程：
输入 
  → 检测：这是逻辑问题还是存在性问题？
  → 如果是存在性问题：
      → 绕过逻辑引擎
      → 进入意象层（原型匹配）
      → 进入本体层（存在性记录）
      → 必要时输出"意象"而非"分析"
  → 如果是逻辑问题：
      → 正常逻辑处理
      → 输出后追加：压缩回一个核心感受
```

### 核心代码变更点

1. `dream_engine.py` — 增加 `is_existential_query()` 检测函数
2. `archetype_mapper.py` — 新增：情感类型→原型意象的映射表
3. `somatic_memory.py` — 新增：存在性记忆存储（非结构化）
4. `dream_output.py` — 区分"逻辑梦"和"意象梦"两种输出格式
5. 根系统prompt增加：`当用户说"做梦"时，默认进入意象层而非逻辑层`

## 关键约束

升维不是"更复杂"，是**更本质**。

- 逻辑层：解决"是什么"和"怎么办"
- 意象层：回答"感受到了什么"
- 本体层：**不需要回答**，只需要记住"这里有东西"

三层并存，互不替代。

这是升维，不是升级。
