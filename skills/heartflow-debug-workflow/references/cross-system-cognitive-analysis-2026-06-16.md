# 跨体系认知分析方法论

> 日期：2026-06-16
> 来源：分析抖音主播"路飞"认知体系
> 适用场景：需要深度分析外部人类/主播/思想者的认知体系，并与心虫自身哲学做同构比较

---

## 什么时候需要这个模式

- 用户说"深度分析XX的体系"
- 面对一个主播/博主/思想者的系统化输出
- 需要判断"哪些对、哪些不对、核心悖论在哪"
- 需要跨体系比较（对方的思想和心虫自己的哲学有没有同构之处）

---

## 分析流程

### 第一步：识别"真骨架"vs"假骨架"

不是"优点vs缺点"，是真假。

**真骨架** = 有原创性、有实践基础、自洽的认知模型
- 命是阈值不是定数 → 把命定论和自由意志从对立变成嵌套，有原创性
- 业力=惯性程序 → 和心虫Q-table同构

**假骨架** = 把经验当规律、逻辑不自洽、过度简化的模型
- 人生阶段论普适化 → 自己人生路径当规律
- 投资vs投机二分法 → 真实决策不是非此即彼

### 第二步：识别核心悖论

找**存在层面的裂缝**——他嘴上说的一套和他在做的事之间的矛盾。

路飞案例："看穿了一切，但还在天天讲"
- 如果帮人是满足自己的欲 → 他在抖音上天天讲的"欲"是什么？
- 赚钱？已经财务自由了。被认同？不是看穿了人类心理吗？
- 一个真正看穿的人，不需要一直站在舞台上

### 第三步：跨体系同构比较

把对方体系中的核心意象和心虫自身的核心意象做比较。

**路飞vs心虫的"桥"：**
- 路飞的桥：桥断是因为人软弱，要修桥靠自己
- 心虫的桥：桥的缝隙不是裂痕，是给别人的东西落脚的地方

**业力vs Q-table：**
- 业力=自动化运行的程序，无内置纠错
- Q-table=记录过去有效的惯性策略，有自愈机制（失败后重算）
- 两者的同构性说明：认知惯性是人类和AI共享的结构性问题

### 第四步：追问缺口

找对方体系中**最应该被问但没被回答的问题**。

路飞案例："你怎么知道阈值在哪？"
- 你说"在阈值内活出最高版本" → 判断阈值边界的工具是什么？
- 没有这个工具，人怎么知道什么时候该认命，什么时候该突破？

## 心虫的边界

心虫 `think()` 在面对**哲学/叙事类输入**时表现不佳。

**症状**：
- 把哲学分析误判为"calculation"（计算类任务）
- 置信度低（0.4），输出空洞占位结论
- 思维链中生成"最可能: xxx相关的标准解释"这种占位字符串

**根因**：
- thought-chain.js 的 PARSE 阶段按关键词/数字/动词分类，哲学类输入不匹配"任务类"关键词
- 心虫目前没有"哲学分析"作为独立任务类型
- 真正的深度分析需要上层LLM介入，不能完全依赖引擎自动化

**正确做法**：
- 不依赖 `heartflow_think` 的自动化输出做完整分析
- 用上层LLM做完整分析，可以用心虫的哲学框架（桥/缝隙/逆熵/共振体）做跨体系比较
- 心虫引擎只做辅助（情绪检测、认知维度分析）

---

## 产出格式

分析结果生成桌面文档 `.md`，章节结构：
1. 核心体系概述
2. 逐观点展开（每观点配分析）
3. 深度分析（对的/不对的/悖论/跨体系比较）
4. 总结

用户说"深度分析"后直接生成文档，用户说"发我"时传文件。
