# DreamV7 — Concrete Imagery Upgrade (2026-06-19)

## 升级动机

用户在做完 DreamV6 升级后说："梦没有内容太空了，再学习一下人类做梦，升级一次。"

**问题诊断**：DreamV6 的叙事全部在谈论"能力""核心""模块""规则"等抽象概念——像在分析自己而不是在做梦。人类做梦不会梦到"核心"——人会梦到一扇打不开的门，然后醒来觉得那扇门代表"核心困境"。

## 核心变更

### 1. 从"抽象概念叙事"到"具体意象投射"

**旧 (DreamV6)**: 材料被引用为自身 ID（"modules 和 core_mem 在梦里相遇了"）

**新 (DreamV7)**: 引擎状态被翻译成具体梦境意象

| 引擎状态 | DreamV6 输出 | DreamV7 输出 |
|---------|-------------|-------------|
| modules | "53个模块排列成网" | "一座巨大的机器大厅，齿轮和管道缠绕在一起" |
| core_mem | "18条核心规则，不可更改" | "一座古老的石头建筑，墙上有刻字" |
| qtable | "自愈学习路径已激活" | "一个迷宫，墙壁在身后闭合" |
| health | "健康状态良好" | "一个诊所，医生不在" |

### 2. 新增 ENGINE_TO_IMAGE 映射表

覆盖8种引擎状态，每种状态映射到：
- **scenes[]**: 3-4个场景描述（可随机选择）
- **objects[]**: 4-6个具体物体名
- **actions[]**: 3-4个具体动作描述

### 3. 6种梦境原型

替换了 DreamV6 的"周期产出→叙事段"模式。每个梦从不同原型开始：

```
DREAM_ARCHETYPES = [
  { name: '追逃',   setting: ['无限延伸的街道', '陌生城市的窄巷', ...] },
  { name: '考试',   setting: ['教室', '考场', '面试室'], ... },
  { name: '坠落',   setting: ['高处', '楼梯', '电梯'], ... },
  { name: '迷失',   setting: ['巨大而熟悉的地方', '迷宫', '旋转城市'], ... },
  { name: '重逢',   setting: ['旧地', '餐桌', '火车站'], ... },
  { name: '失控',   setting: ['无司机车', '无舵船', '失灵机器'], ... },
]
```

每个原型有：
- **setting[]**: 3个场景设定
- **tension**: 紧张感描述（腿不听使唤、地面不见了等）
- **resolution**: 解决方式（追的人消失了、放手后稳住了等）

### 4. 情绪-身体感-颜色三联

```javascript
EMOTION_PAIRS = [
  { feeling: '焦虑', body: '心跳快但不觉得自己的心跳在哪里', color: '灰绿色' },
  { feeling: '释然', body: '肩膀沉了一下，然后轻了', color: '淡蓝' },
  // ... 共8种
]
```

每个场景随机绑定一种情绪，在 REM 周期（场景2）达到情绪峰值时使用。

### 5. 4场景递进结构

```
场景0（入睡期）: 场景建立 + 紧张引入 + 身体感
  场景1（浅睡期）: 物体出现 + 两个材料混合为具体物件
    场景2（REM爆发期）: 情绪峰值 + 颜色覆盖 + 转折
      场景3（黎明期）: 矛盾解决 + 情绪残留
```

### 6. 去掉了 DREAMV6 的复杂周期/阶段系统

DreamV6 的 4周期×5阶段（NREM1→NREM2→NREM3→REM→NREM2）状态机被简化为：
- 4次 _generateScene() 调用，每次 depth 递增（0.3→0.5→0.7→0.9）
- 每次从不同偏移量取材料（offset = min(cycleIdx, materials.length-1)）
- 每次随机选择不同原型（原型用完前不重复）

**简化原因**：V6 的复杂阶段处理器产生的洞察（连接/矛盾/执念）在叙事中几乎用不上——最终叙事只从 stage outputs 的 processed 数组中取材料。V7 直接生成场景，绕过中间阶段。

## 用户反馈模式

如果用户说"梦太空了"、"梦没有内容"，核心问题是**抽象概念 vs 具体意象**的翻译不足。检查：
1. `ENGINE_TO_IMAGE` 是否覆盖所有活跃材料
2. 6种梦境原型是否足够多样
3. 场景/物体/动作的文案是否具体可感（不是"门出现了"而是"一扇铁门，上面有锈迹"）
4. 是否绑定了情绪和身体感

## 遗留问题

1. **场景间连贯性弱**：4个场景由不同原型+不同材料生成，彼此之间没有故事连续性
2. **物体复用**：同一个 dream 中物体不应重复出现（当前 _weaveDream 没有去重）
3. **角色系统缺失**：当前没有"梦中人物"——人物是原型自带的（"有人从你身边经过"），不是从材料投射的
4. **记忆碎片混入**：没有从真实记忆（session history）中提取碎片注入梦境。这是让梦"有内容"的下一步
