# Upgrade Report — v2.8.22

**Date**: 2026-06-09
**Module**: `src/memory/knowledge-graph.js` (5274B → 17838B)
**Version**: 1.0.2 → 1.1.0

## What was upgraded

KnowledgeGraph — graph/network engine class for structured memory with temporal context.

### Added features

1. **Input validation layer** — 6 safe utility methods (`_safeString`, `_safeNumber`, `_validateNodeInput`, `_validateEdgeInput`) for all public methods. Null/type/empty checks with descriptive error messages.

2. **KG_ERROR enum** (6 types) — `NODE_NOT_FOUND`, `EDGE_NOT_FOUND`, `CONNECTION_LIMIT`, `INVALID_INPUT`, `SEARCH_EMPTY`, `UNKNOWN` with `KG_ERROR_MESSAGES` lookup.

3. **Node removal** (`removeNode`) — removes node and all associated edges, cleans up connection lists on other nodes.

4. **Edge weight update** (`updateEdgeWeight`) — update edge weight with bounds clamping.

5. **Enhanced search** — returns scored results (`score`, `rankScore = score * importance`) sorted by rankScore, not just importance.

6. **findByName** — exact name-based node lookup.

7. **Importance decay** (`_applyDecay`) — time-based decay (0.95/day), faster decay for nodes older than 30 days, minimum floor at 0.05.

8. **Stale node pruning** (`pruneStale`) — configurable minImportance/maxAgeMs/minReflections thresholds, returns list of removed nodes.

9. **JSON serialization** (`toJSON`/`fromJSON`) — full graph state export/import with field-level validation on restore.

10. **Enhanced statistics** — `staleCount`, `edgeTypeDistribution`, `nodeTypeDistribution`, `mostReflected` (top 5), `isDirty` flag.

11. **Defensive programming** — all public methods validate inputs, throw typed errors or return safe defaults.

### Backward compatibility
- All original public methods keep same signatures
- `search()` now returns scored objects `{node, score, rankScore}` instead of raw nodes — but heartflow.js only uses `getStats()` from this module
- `getConnectedNodes()` now returns `{node, edgeWeight, relation}` objects instead of raw nodes — only referenced by heartflow.js, which doesn't call this method directly
- Only 1 reference point (`src/core/heartflow.js`), verified working
