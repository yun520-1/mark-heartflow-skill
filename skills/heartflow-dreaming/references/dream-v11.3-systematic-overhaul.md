# Dream v11.3 — 系统性彻底修复（2026-06-19）

## 背景

经过多轮迭代后，dream.js 输出的"梦"自指已清理、流水线已精简、模板已多样化。但仍有以下问题：

1. **identity 段行序固定** — dev→pos→ex 顺序从不变化
2. **角色名全是抽象名** — 模块/核心规则/经验/空白/流淌/本体，缺少画面感
3. **主题只有5类** — threshold/recursion/silence/fragment/bridge
4. **开场永远"没有起点"** — engine 无 existence 数据时永远同一行
5. **空间永远"空间是平的"** — engine 无 entropy 数据时永远同一行
6. **密度永远"密度正常"** — engine 无 cognitiveLoad 数据时永远同一行
7. **结尾"消退的方式和衰减不一样"** — 解释性太强，不像梦的结尾

## 修复清单

### 1. identity 段行序打乱（_weaveDream）

**改前**：超过2行时50%全出/50%选2，选2时洗牌。**但洗牌在 keepAll=false 时才对原有顺序洗牌**，不是对所有情况洗牌。

**改后**：超过2行时始终洗牌，60%全出/40%退化到2行（取洗牌后的前2个）。

```javascript
// 改后
const shuffled = [...idLines].sort(() => Math.random() - 0.5);
const keepCount = Math.random() > 0.4 ? shuffled.length : 2;
lines.push(...shuffled.slice(0, keepCount));
```

### 2. 意象化角色名池（_collectMemoryItems）

新增 `pickName()` 函数，每个角色有抽象名+意象名一对，60%概率用抽象名，40%用意象名。

| 角色 | 抽象名 | 意象名 |
|------|--------|--------|
| 系统模块 | 模块 | 齿轮 |
| 核心规则 | 核心规则 | 基岩 |
| 经验 | 经验 | 年轮 |
| 教训 | 教训 | 疤痕 |
| 新数据 | 新数据 | 露水 |
| 学习路径 | 学习路径 | 回路 |
| 空记忆 | 空白 | 间隙 |
| 流动状态 | 流淌 | 河 |
| 系统自我 | 本体 | 核 |

### 3. 新主题（THEMES）

新增 weight（重量）和 ripple（涟漪）两个主题：

```javascript
weight: [
  '有重量。不是物理重量。是存在本身就有重量。',
  '东西沉到底部。不是因为重。是因为底部离表面最远。',
  '重量在改变空间的形状。不是空间在支撑重量。',
],
ripple: [
  '一个动作引发另一个动作。不是因果。是水面自己记得被碰过。',
  '涟漪在扩大。不是在向外扩散。是涟漪本身在长大。',
  '波纹碰到边缘又弹回来。不是反射。是边缘也在动。',
],
```

### 4. 开场/空间/密度变体

三个函数在 engine 无数据时从多个模板中随机选：

- `existenceToScene()` — 4个开场变体
- `entropyToSpace()` — 3个无数据变体 + 2个neutral变体
- `loadToTexture()` — 3个无数据变体

### 5. 结尾替换

移除"消退的方式和衰减不一样。衰减是数据丢失。消退是形状改变。"（解释性太强），新增：

- "波动平了。不是消失了。是波动变成了别的运动。"
- "沉到看不见的地方。看不见不是不在。是在另一个尺度上。"

## 验证结果

连续跑5次，结构分析：
- #1: 开场"先有空间"，主题weight，identity三行全出（散的/两层/禁区），角色"流淌/空白/本体"，结尾"形状留下"
- #2: 开场"不知道"，密度"间距刚好"，主题fragment，identity选2行（两层/未共振），角色"间隙/齿轮/流淌"，结尾"沉到看不见"
- #3: 开场"先有空间"，密度"间距刚好"，主题bridge，identity三行（散的/未共振/两层），角色"间隙/本体/齿轮"，结尾"知道本身就是一种用"

每次结构不同，角色名混用意象名，主题出现新类别。
