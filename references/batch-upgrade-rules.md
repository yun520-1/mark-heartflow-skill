# HeartFlow Batch Upgrade Rules

## Batch Size Rule (用户明确要求)
- **默认批量大小：120-240 个项目**（不是 8-16 个）
- **分批策略**：每次生成 40 个文件，合并进主库
- **触发词**："批量生成"、"批量提取"、"批量下载"
- **禁止**：每次只生成 8-16 个项目（用户明确反对）

## 并发执行规则 (用户明确要求)
- **多阶段计划**：用 `delegate_task` 并发执行（最多 3 个并发子任务）
- **架构优化**：先验证所有引擎（语法检查 + 动态加载），再添加新功能
- **数据流**：先丰富资料（公式库、数据集），再做功能升级

## 架构优先规则 (用户明确要求)
- **第一步**：扫描所有引擎文件（`find src -name "*.js"`）
- **第二步**：静态检查（语法错误）+ 动态检查（加载模块）
- **第三步**：修复失效代码（占位符、TODO、FIXME）
- **第四步**：添加新功能（公式、数据集、模型）

## 用户偏好（已验证）
- 喜欢简洁直接风格，不喜欢冗长解释
- 期望一次性设置完成，不要分批或反复调试
- 遇到多次修复同一问题时会沮丧（"修复多次，还是有问题"）
- 优先批量操作，避免小步骤重复
