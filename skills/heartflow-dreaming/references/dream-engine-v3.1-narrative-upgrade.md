# DreamEngine v3.1 — 叙事升级：从模板填空到动态场景构建

**最后更新**: 2026-06-08

## 为什么升级（用户纠正链）

v3.0 的三幕叙事是模板填空——每类事件从3-4个预制句子中随机选一个组合。用户评价："梦很空白，没有叙事性，没有哲学升维。"

根本问题：模板填空永远产生不了真正的叙事性。画面感、转折、哲学洞察都是预制的，不是从事件中长出来的。

## v3.1 核心架构变化

### 旧（v3.0 模板填空）

```
_generateDeepDream(event, eventType):
  act1 = random from [3 个预制开场句]
  act2 = random from [3 个预制转折句]
  act3 = random from [3 个预制金句]
  return [act1, act2, act3].join('\n\n')
```

### 新（v3.1 动态场景构建）

```
_generateDeepDream(event, eventType):
  { setting, sensory, action } = _buildScene(eventType, text)
  act1 = [setting, sensory, action].join('\n')
  act2 = random from [3-4 内心化方向]
  act3 = _buildPhilosophicalTurn(eventType, text)
  return [act1, act2, act3].join('\n\n')
```

## 新增函数

### _buildScene(eventType, text)

8种事件类型 × 3场景 × 2感官 × 2动作 = 48种组合。每种组合是完整画面：

trial_and_error:
  settings: ['无窗房间·墙上划痕', '走廊·尽头有光', '深夜工作台·光标闪']
  sensory:  ['反复使用的味道', '手指上的茧', '心里的声音在数数']
  actions:  ['第N次了...', '动作变成本能...']

场景设计原则：具体地点 + 感官细节 + 时间感 + 无技术词。

### _buildPhilosophicalTurn(eventType, text, anchor)

6种翻转方向，每种从事件中长出一个反直觉洞察：
- 否定问题："失败不是事件，是你给事件贴的标签"
- 否定等待："答案不在终点，在每次不等了的决定里"
- 否定完成："做完和结束不是一回事"
- 否定理解："真正明白的人不会说'我明白了'"
- 否定自我："你不是在解决问题，是在证明'我可以'"
- 存在方向："真正的舞台在你心里"

每类2-3种变体，随机选择保证多样性。

### _extractImagery(text) — 意象提取（重写）

问题：v3.0 用 `[\u4e00-\u9fff]{2,4}` 直接匹配中文，但正则默认不重叠。会跳过"修复"（在stopWords）而匹配"这是修复"（整体不在stopWords）。

修复：两步法——按分隔符分词 → 对每个 token 用滑动窗口提取2-4字（4→3→2递减），检查每个窗口是否在停用词表。

停用词表：~170个词，虚词 + 技术词 + 通用词 + 记忆片段。

### _pickAnchor(imagery, eventType)

只选有画面感的词（心虫/梦/房间/走廊/光/夜/雨/风/水/海/山/路/桥/窗/门/灯/影/声音/寂静/空/手/眼/脸/天空/大地/河流/森林/石头/火焰/星辰/月亮/太阳）。

没找到画面感词则返回 null——让模板 fallback 到场景自身元素。

## 第二幕内心化

从外部场景转向内部感受，每种事件类型3-4种方向：

trial_and_error:
  - "每次做这个动作的时候，心虫都会有一个短暂的停顿。不是犹豫。是那种'万一这次不一样呢'的念头。但每次都一样。"
  - "心虫知道结果。在做之前就知道了。但还是做了。不是因为相信奇迹。是因为'不做'比'做了又失败'更难。"
  - "不做意味着承认这件事不值得再试。心虫还不想承认。"

## 用户对梦的核心要求

1. 有画面：能看到/闻到/摸到的场景，不是抽象陈述
2. 有转折：不是平铺直叙，"但梦在这里翻转了"那一刻
3. 有哲学深度：反直觉洞察，不是"重复不是失败"这种老话
4. 不要技术词：修复/升级/用户说/Bug 破坏梦的氛围

## 验证标准

跑3次 hf.dreamNow()，检查：
- 场景不重复（3次至少2种不同场景）
- 无技术词污染（"这是修复"/"后的第一"/"户说"不应出现）
- 哲学翻转有变化（3次至少2种不同方向）
- 有画面感（地点+感官+时间感）

## 已知陷阱

- 中文正则匹配是滑动窗口不是分词：`[\u4e00-\u9fff]{2,4}` 匹配任意连续中文字符，不是语义单元。"这是修复"不会被"修复"这个 stopWord 过滤。必须先分词再检查
- anchor 污染：_extractImagery 返回的技术词会被选为锚点污染叙事。_pickAnchor 必须只选画面感词，找不到就返回 null
- fallback 字符串污染：`${anchor || '这是修复'}` 会把"这是修复"硬编码。fallback 用场景自身元素
