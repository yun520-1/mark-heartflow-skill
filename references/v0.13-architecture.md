# HeartFlow v0.13 新架构规范

> 重建目标：功能齐全、运行顺畅、安全健壮、安装简单的 AI Agent 框架

## 1. 设计原则

### 1.1 核心目标
- **让全世界 AI 安装使用** — 安装简单，零配置即可运行
- **功能齐全** — 覆盖 AI Agent 全生命周期（记忆、推理、决策、执行、进化）
- **运行顺畅** — 无冗余、无死锁、无内存泄漏
- **可升级** — 支持未来 AI 能力升级

### 1.2 架构参考
- **LangGraph** — 状态机+checkpointing，支持循环
- **AutoGen** — 多智能体对话编排
- **Reflexion** — 自我反思模式
- **三层记忆架构** — Working/Episodic/Semantic Memory

### 1.3 设计准则
1. **强类型 TypeScript** — 所有接口显式声明
2. **纯函数 + 不可变数据** — 易于测试和推理
3. **事件驱动** — 引擎间通过事件总线通信
4. **错误边界** — 每个引擎独立错误处理
5. **Checkpointing** — 定期保存状态，支持恢复

## 2. 系统架构

### 2.1 层次结构

```
CLI / API Layer
       ↓
Orchestrator Layer (Supervisor)
       ↓
Agent Layer (Ethics/Autonomy/Consciousness/Transmission)
       ↓
Core Engine Layer (Identity/Memory/Evolution/Cognition)
       ↓
Runtime (event-bus / config)
```

```

├── core/
```

## 3. 核心接口

### HeartFlowEngine

```typescript
interface HeartFlowEngine {
  version: string;
  boot(): Promise<void>;
  shutdown(): Promise<void>;
  ready: boolean;
  identity: IdentityEngine;
  memory: MemoryEngine;
  evolution: EvolutionEngine;
  cognition: CognitionEngine;
  ethics: EthicsEngine;
  autonomy: AutonomyAgent;
  consciousness: ConsciousnessAgent;
  transmission: TransmissionAgent;
  run(input: {task: string; context?: Record<string, unknown>}): Promise<AgentOutput>;
}
```

### 各引擎接口

- **IdentityEngine**: boot() → IdentityStateData, getCoreIdentity(), evolve()
- **MemoryEngine**: boot(), working/episodic/semantic 三层, importMemory(JSON字符串), recall(key), shutdown()
- **EvolutionEngine**: boot(), evolve(population, config), getStats(), shutdown()
- **CognitionEngine**: boot(), reason(input), plan(goal, context), reflect(events), shutdown()
- **EthicsEngine**: boot(), judge({action, context}) → EthicalJudgment, shutdown()
- **AutonomyAgent**: boot(), decide(context) → AutonomyDecision, getConfig(), shutdown()
- **ConsciousnessAgent**: boot(), getState(), thoughtArising(), selfReflect(), getRecentEvents(), shutdown()
- **TransmissionAgent**: boot(), send(target, action, data), sendWithRetry(), getQueue(), shutdown()

## 4. 编排流程（Supervisor 模式）

1. Identity check — 身份确认
2. Memory recall — 记忆检索
3. Cognition reasoning — 推理计划
4. Ethics review — 伦理审查（rejected 则终止）
5. Autonomy decision — 自主决策
6. Result — 结果返回
