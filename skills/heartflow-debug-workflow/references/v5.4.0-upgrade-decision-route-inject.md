# v5.4.0 决策路由注入 + 置信度区分度 + 英文推理增强 升级参考

## 升级内容

### 1. pipeline decision 阶段注入9字段

文件：`src/workflow/pipeline.js`

**之前**：只传5个字段（inputText/cognitiveLoad/directionClear/confidence/dissonance）
**之后**：注入9个字段（新增 quality/stability/severity/identityCoherence/desireDominant/poisonLevel）

**关键修改**：
- cognitiveLoad 从多源汇聚：`agentPsych.cognitiveLoad.load` → `agentPsych.cognitiveLoad` → 0
- directionClear 动态计算：`direction === 'act' ? 0.8 : direction === 'analyze' ? 0.6 : direction ? 0.4 : 0.3`
- dissonance 从 goalConflicts/cognitiveDissonance 汇聚
- severity 从 painLevel 映射
- 新增 logicReasoning 注入到 judgment context

### 2. 记忆系统回填

**之前**：`hf.memory.store('learned', ...)` 把 'learned' 当 key 用，不是写入 learned 层
**之后**：`hf.memory.store('judgment:${Date.now()}', ...)` 正确写入，自动根据前缀分配层

**CORE seeding**（首次运行时写入）：
- `identity.engine_role`：引擎是底层认知分析系统
- `identity.decision_principle`：决策基于结构化数据
- `identity.confidence_principle`：置信度必须有区分度

### 3. 英文推理信号增强

文件：`src/reasoning/logic-reasoning.js`

**之前**：deductive minKeywords=3，无数学运算关键词，大小写敏感
**之后**：minKeywords=2，新增30+数学词（calculate/total/sum/difference/find/solve/how many等），大小写不敏感

### 4. 置信度区分度

文件：`src/core/judgment-engine.js`

**之前**：analyze 路径恒为 6.0/10 → confidence=0.6
**之后**：基于输入长度/情绪强度/谬误检测/意图明确度动态调整

### 5. _matchKeywords 大小写不敏感

**之前**：`input.includes(kw)` — "Therefore" 不匹配 "therefore"
**之后**：`input.toLowerCase().includes(kw.toLowerCase())`
