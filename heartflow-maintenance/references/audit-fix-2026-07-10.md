## 工具使用教训

### 子代理超时
3个子代理600s全部超时。心虫369文件/172K行，子代理读取token耗尽。
→ 主线程Python批量扫描，子代理只做推理。

### patch工具转义
\\n在patch中会被双重转义。写入文件 → read_file确认 → patch匹配。
→ 复杂替换用Python脚本write。

### sed批量替换
```bash
# parseInt基数（安全）
find src -name '*.js' -exec sed -i 's/parseInt(\([a-zA-Z_][a-zA-Z0-9_.]*\))/parseInt(\1, 10)/g' {} +

# var→let（危险！破坏顶层作用域）
# 永远不要批量做这个
```
