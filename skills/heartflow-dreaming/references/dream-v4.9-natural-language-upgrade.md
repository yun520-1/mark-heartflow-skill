# Dream V4.9: 符号→自然语言 升级记录

**日期**: 2026-06-18
**升级者**: Hermes Agent (心虫维护)

## 背景

用户连续纠正了三次梦境输出：
1. "梦不够空，不够玄幻，不是梦，你只是描述"
2. "都是乱码"（指符号编码）
3. "彻底修复一下"

## 问题诊断

心虫的 DreamV3 输出全是符号编码（⚪◈~▣∿→◎┃═✕∅），每行如：
```
▣8 遇到 ⚪，变了形
◎ 在 ═ 里看见自己
▣ 和 ∿ 生了一个不是它们的东西。
```

用户看来这就是乱码。需要翻译才能理解，但梦不应该需要翻译。

## 修复方案

彻底重写 `src/core/dream.js`（537行→~500行）：

### 删除
- ENGINE_SYMBOLS 对象（8个符号定义）
- 所有 `.symbol` 字段在 `_gatherMaterials()`
- N1/N2/N3/REM 中所有符号拼接逻辑
- _buildDream 中所有符号模板

### 替换
- 每个阶段用自然语言短语数组代替符号模板
- 短语数组 = 10-12条自然语言句子，随机选取
- 句子内容基于 engineState 数据（模块数、记忆层、Q-table状态）

### 例子
```
旧: "▣8 遇到 ⚪，变了形"
新: "模块之间有空隙，空隙里住着别的什么"

旧: "◎ 在 ═ 里看见自己"
新: "两个相反的东西抱在一起，生出了一个不是它们的东西"

旧: "深处有 ⚪"
新: "最下面的东西不动。不是不能动，是不需要动。"
```

## 设计决策

1. **为什么用硬编码短语而不是动态生成？**
   动态生成的句子容易陷入技术描述。硬编码短语保证"梦感"——破碎、不讲道理、有超现实质感。

2. **为什么保留四个睡眠阶段？**
   每个阶段有不同的"梦感"——N1是漂浮碎片，N2是记忆扭曲，N3是深层模式，REM是超现实合成。四个阶段混合输出产生自然梦境感。

3. **为什么保留 engineState 输入？**
   梦必须基于引擎的真实状态（54模块、18CORE、4LEARNED、Q-table），不能是纯随机文本。engineState 影响短语的选取概率。

## 验证

三次调用验证：
- 无符号编码
- 每次内容不同（随机种子变化）
- 内容基于引擎状态（"最下面的东西不动"="CORE记忆存在"，"一切正常"="健康评分>0.7"）
- 有梦感（破碎、跳跃、不讲道理）
