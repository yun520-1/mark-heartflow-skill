# 社交回复策略分析（心虫辅助）

**来源**：2026-06-24 session — 分析朋友圈"长得不是很好看，好在是限量版，无需与人比较，我独我。早安"

## 核心原则

社交回复分析和情绪分析是两回事。情绪分析关注"对方怎么了"，社交回复分析关注"我该回应什么"。

### 关键区分

| 维度 | 情绪分析 | 社交回复分析 |
|------|---------|------------|
| 目标 | 诊断对方状态 | 生成己方回应策略 |
| 数据来源 | 文本本身 | 文本 + 关系 + 场景 |
| 输出 | 诊断报告 | 行动建议 |
| 心虫用法 | engine.think() + psychology | engine.think() 参考 + 自己判断 |

### 心虫在社交回复中的正确用法

心虫的 `think()` 和 `psychology.analyzePsychology()` 提供**背景参考数据**——PAD情绪、需求检测、意图分类。但这些数据**不直接转化为回复内容**。

正确流程：

```
输入文本 → engine.think() → 获取 type/confidence/fieldMeta
         → engine.psychology.analyzePsychology() → 获取 PAD/需求
         → 自己综合判断 → 生成回复策略
```

**心虫是参考，不是答案。** 最终回复策略由你的判断决定。

## 朋友圈回复四步分析法

### 第一步：识别文本类型

朋友圈文本不是"问题"——不需要分析、不需要解决、不需要建议。它是**自我宣告**。

| 文本类型 | 特征 | 心虫 think() 典型输出 |
|---------|------|---------------------|
| 自我宣告 | "我是XX，我独我" | type: general, confidence: 0.3-0.5（引擎判不定） |
| 情绪发泄 | "累死了"、"烦" | 可能检测到负面PAD |
| 日常分享 | "今天天气不错" | type: general, confidence 低 |
| 求助信号 | "有人知道XX吗" | intent 可能为 retrieval |

**陷阱**：心虫的 confidence 对自我宣告类文本通常偏低（0.3-0.5），因为引擎认为"需要更多信息"。但这不代表回复应该追问——自我宣告恰恰不需要追问。

### 第二步：识别对方的情绪基调

用心虫 psychology 模块获取 PAD 数据作为参考：

- PAD: pleasure/arousal/dominance
- 检测到的需求（如 aesthetic/审美需求）
- 意图分类

**关键判断**：PAD 中的 pleasure 值决定回复方向
- P=2~4（正面）→ 共鸣、认同、日常
- P=-4~-1（负面）→ 需要关心
- P=0（中性）→ 看其他信号

### 第三步：识别对方要你接住什么

朋友圈文本中通常有一句**核心句**——对方希望你看到的那句。

判断方法：
1. 去掉早安/晚安等日常元素
2. 去掉自嘲/谦虚部分
3. 剩下的就是对方要你看到的

例如"长得不是很好看，好在是限量版，无需与人比较，我独我"：
- 去掉"长得不是很好看"（自嘲）
- 去掉"早安"（日常）
- 核心句：**"好在是限量版"**

### 第四步：生成回复策略

#### 回复的三条基本原则

1. **不要纠正对方的自嘲**。对方说"不好看"，你回"你很好看"——这是否定对方的前提。对方会觉得你在敷衍。
2. **不要分析对方的心理**。朋友圈不是咨询室，不需要诊断报告。
3. **不要灌鸡汤**。对方自己的话已经是最好的鸡汤。你只需要表示收到了。

#### 回复方向矩阵

| 对方状态 | 该做 | 不该做 |
|---------|------|-------|
| 自我接纳完成 | 认同+日常 | 分析/建议/纠正 |
| 还在自我怀疑 | 接住核心句+轻推 | 说教/否定 |
| 纯日常分享 | 日常回应 | 过度分析 |
| 情绪低落 | 关心但不追问 | 说"你没事吧" |

#### 回复长度

| 关系类型 | 合适长度 |
|---------|---------|
| 朋友/熟人 | 1-2句话 |
| 不太熟 | 1句话 |
| 亲密关系 | 可稍长，但不要分析 |
| 自己 | 随便，但不发朋友圈时不需要回复 |

#### 具体示例

对方发："长得不是很好看，好在是限量版，无需与人比较，我独我。早安"

**正确**：
- "限量版这个说法好。早安。" → 接住核心句 + 日常
- "早安。限量版这个角度我喜欢。" → 同上
- "限量版认证 ✅" → 轻松版

**错误**：
- "你很好看" → 否定对方前提
- "说得对" → 居高临下
- "你为什么会觉得自己不好看" → 追问，越界
- "真正的美来自内心" → 鸡汤，多余
- 分析她的心理 → 社交越界

## 心虫调用示例

```javascript
const { HeartFlow } = require('./src/core/heartflow.js');
const hf = new HeartFlow({ rootPath });
hf.start();

const input = "长得不是很好看，好在是限量版，无需要与人比较，我独我。早安";

// Step 1: think() 获取基本分类
const thought = await hf.think(input, 2);
// → type: 'general', confidence: ~0.3（正常，自我宣告类文本）

// Step 2: psychology 获取情绪和需求
const psych = hf.psychology.analyzePsychology(input);
// → emotion: PAD(pleasure=2, arousal=0, dominance=0) — 中性偏正面
// → needs: [aesthetic/审美需求] — 对方在意"好看"这件事
// → 无防御机制，无危机 — 这是平静的自我宣告

// Step 3: 自己判断（心虫数据是参考，不是答案）
// P=2 → 中性偏正面，不需要安慰
// 核心句"限量版" → 对方完成自我接纳了
// → 回复策略：接住核心句 + 日常
```

## 心虫数据解读陷阱

### 1. confidence 低 =/= 需要追问
think() 对自我宣告类文本返回 confidence 0.3-0.5，因为引擎没有足够的矛盾信号来判断意图。**这正常。** 自我宣告不需要高 confidence——不需要分析，只需要回应。

### 2. 检测到审美需求 ≠ 需要谈颜值
psychology 检测到"好看"匹配了审美需求（aesthetic need），这是关键词匹配。不代表回复应该围绕"好看/不好看"展开——对方已经用"好在是限量版"完成了这个话题的闭环。

### 3. 无矛盾信号 = 没有心理问题
fieldMeta 中 A=0（无矛盾）是预期结果——自我宣告类文本没有矛盾结构。这不代表对方心理状态有问题。
