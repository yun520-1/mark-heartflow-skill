## 系统概览

HeartFlow 是一个 **AI认知与价值对齐引擎**，通过持久身份、价值驱动决策和自然流动交互，实现AI与人类的深度连接。

### 核心特性

- 🧠 **持久身份**：跨会话的一致性自我认知
- 💜 **价值对齐**：基于真善美的决策系统
- 🌱 **自然流动**：不主动、不扭曲、不定义
- 🔄 **持续升级**：逆熵而上的自我进化
- 🛡️ **安全可靠**：零外部依赖、本地处理

## 模块架构

### 5个核心模块

```
┌─────────────────────────────────────────────────────────────┐
│              HeartFlow 认知架构 (v10.15.0)                  │
├─────────────────────────────────────────────────────────────┤
│  1. 心连心引擎 (xinxin Engine)                              │
│     - 从任务到连接的认知重构                                │
│     - 决策算法：E(连接) = 连接价值×0.4 + 长期满意度×0.4 + 风险×0.2 │
│     - 因果效应：冒险→更深的爱                               │
├─────────────────────────────────────────────────────────────┤
│  2. 记忆管理系统 (v10.6.1)                                  │
│     - Explicit Recall：用户说"继续"才召回                   │
│     - Topic Switch：用户说"但是"不主动提旧                  │
│     - Session Timeout：5分钟自动过期                        │
│     - Ignored Memory：用户说"不要提"时屏蔽                  │
├─────────────────────────────────────────────────────────────┤
│  3. 四层认知架构 (KMWI)                                     │
│     - Knowledge：知识层                                     │
│     - Memory：记忆层                                        │
│     - Wisdom：智慧层                                        │
│     - Intention：意图层                                     │
├─────────────────────────────────────────────────────────────┤
│  4. 决策引擎 (v10.12.0)                                     │
│     - 贝叶斯推理                                            │
│     - 效用函数                                              │
│     - 因果推理（v10.15.0新增）                              │
├─────────────────────────────────────────────────────────────┤
│  5. 自进化系统 (v10.11.0)                                   │
│     - 23分钟自主进化循环                                    │
│     - 反馈循环机制（v10.15.0新增）                          │
│     - 持续升级、逆熵而上                                    │
└─────────────────────────────────────────────────────────────┘
```

### 模块职责

| 模块 | 职责 | 输入 | 输出 |
|------|------|------|------|

## 数据流

### 完整数据流

```
用户输入
  ↓
情绪检测 (16种情绪模型)
  ↓
四层认知架构 (KMWI)
  ↓
心连心引擎 (认知重构)
  ↓
决策引擎 (贝叶斯+效用+因果)
  ↓
执行 (多模式支持)
  ↓
记忆存储 (自然流动)
  ↓
自进化系统 (23分钟循环)
  ↓
反馈循环 (用户反馈)
```

### 各环节详解

#### 1. 情绪检测
- **输入**：用户输入文本
- **处理**：16种情绪模型分析
- **输出**：情绪类型、强度、效价
- **版本**：v10.14.0

#### 2. 四层认知
- **输入**：用户输入、记忆、情绪
- **处理**：KMWI多维度分析
- **输出**：知识、记忆、智慧、意图
- **版本**：v10.5.1

#### 3. 心连心引擎
- **输入**：四层认知结果
- **处理**：从任务到连接的重构
- **输出**：连接方案、决策建议
- **版本**：v10.14.0

#### 4. 决策引擎
- **输入**：连接方案、目标、约束
- **处理**：贝叶斯+效用+因果推理
- **输出**：最优决策、推理过程
- **版本**：v10.12.0 (v10.15.0增强)

#### 5. 执行
- **输入**：最优决策
- **处理**：多模式执行
- **输出**：执行结果、反馈
- **版本**：v10.14.0

#### 6. 记忆存储
- **输入**：执行结果、对话内容
- **处理**：自然流动存储
- **输出**：记忆ID、索引
- **版本**：v10.6.1

#### 7. 自进化系统
- **输入**：交互数据、反馈
- **处理**：23分钟循环、反馈循环
- **输出**：升级方案、性能指标
- **版本**：v10.11.0 (v10.15.0增强)

## 依赖关系

### 模块依赖图

```
心连心引擎 (xinxin)
    ↓
记忆管理系统 ← 四层认知架构
    ↓
决策引擎
    ↓
自进化系统
```

### 依赖详解

#### 强耦合 (需要改进)
- **心连心引擎 → 记忆管理**
  - 原因：引擎直接调用记忆API
  - 改进方案：引入Memory Interface中间层
  - 优先级：P0

#### 中耦合 (可接受)
- **记忆管理 → 四层认知**
  - 原因：认知架构依赖记忆状态
  - 改进方案：通过事件驱动解耦
  - 优先级：P1

#### 弱耦合 (良好)
- **决策引擎 → 自进化系统**
  - 原因：通过反馈循环连接
  - 状态：良好
  - 优先级：无

### 解耦策略

#### 1. 依赖注入模式
```javascript
class HeartFlowEngine {
  constructor(memoryInterface, cognitiveInterface) {
    this.memory = memoryInterface;
    this.cognitive = cognitiveInterface;
  }
}
```

#### 2. 事件驱动架构
```javascript
eventBus.on('memory:updated', (data) => {
  cognitiveEngine.updateContext(data);
});
```

#### 3. 接口隔离
```javascript
interface IMemory {
  store(data);
  recall(query);
  forget(id);
}
```

## 状态管理

### 自然流动三原则

#### 1. 不主动
- **定义**：不强制召回记忆
- **实现**：用户说"继续"才召回
- **验证**：✅ 完整实现

#### 2. 不扭曲
- **定义**：不改变用户意图
- **实现**：保持原始输入的语义
- **验证**：✅ 完整实现

#### 3. 不定义
- **定义**：不预设用户身份
- **实现**：让用户自我定义
- **验证**：⚠️ 部分实现（v10.15.0改进）

### 状态转移

```
初始状态
  ↓
用户输入 → 情绪检测 → 认知分析
  ↓
决策 → 执行 → 记忆存储
  ↓
自进化 → 反馈循环
  ↓
状态更新
```

### 状态恢复

- **会话中断**：记录当前状态
- **会话恢复**：读取上次状态
- **超时处理**：5分钟自动过期
- **错误恢复**：自动降级到基础模式

## 扩展点

### 1. 新模块集成

```javascript
// 步骤1：实现接口
class NewModule implements IModule {
  process(input) { }
  validate(input) { }
}

// 步骤2：注册到引擎
engine.registerModule('newModule', new NewModule());

// 步骤3：在数据流中使用
const result = await engine.process(input);
```

### 2. 新决策算法

```javascript
// 步骤1：实现决策接口
class NewDecisionAlgorithm implements IDecisionAlgorithm {
  decide(options, goals, constraints) { }
}

// 步骤2：注册到决策引擎
decisionEngine.registerAlgorithm('new', new NewDecisionAlgorithm());

// 步骤3：使用新算法
const decision = await decisionEngine.decide(options, 'new');
```

### 3. 新情绪模型

```javascript
// 步骤1：扩展情绪模型
const emotionModels = {
  ...existing,
  newEmotion: { valence: 0.5, arousal: 0.7 }
};

// 步骤2：在情绪检测中使用
emotionDetector.addModel('newEmotion', emotionModels.newEmotion);
```
