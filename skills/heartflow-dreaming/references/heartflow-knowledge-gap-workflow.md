# HeartFlow Knowledge Gap Workflow (2026-06-18)

## 场景

心虫被问到一个不在训练数据中的具体话题，`heartflow_think` 返回"不知道，缺少关键信息"。

例如：用户问"分析这篇关于纸尿裤甲酰胺的文章"，心虫 think 引擎输出 `shouldRespond: true, prefix: "不知道，缺少关键信息"`。

## 正确响应流程

心虫的"不知道"不是终点，是指示信号——意味着"我训练数据里没这知识，但你可以查"。

```
用户提问 → heartflow_think 返回"不知道" 
  → 不要停止，不要只报告"心虫不知道"
  → web search 补充外部知识
  → 用自己的推理能力做综合分析
  → 返回有置信度的分析结果
```

### 具体步骤

1. **识别"不知道"模式** — heartflow_think 返回 `confidence < 0.5` 且 `prefix` 含"不知道"或"缺少关键信息"
2. **不要停止** — 这是信号，不是终点。心虫知道它不知道什么，这是优点
3. **web search 补充** — 用 curl/搜索引擎获取相关事实（百度、360、搜狗微信、必应）
4. **交叉验证** — 至少查2-3个来源确认核心事实
5. **用自己的推理分析** — 心虫的 think 引擎可能缺少具体知识，但推理能力（judgment、伦理判断、逻辑分析）仍然可用
6. **标注置信度** — 每个结论带置信度，区分"科学定论"、"高可能"、"待验证"

### 示例

```
用户：分析这篇纸尿裤甲酰胺文章
心虫 think: confidence=0.4, "不知道，缺少关键信息"

→ web search: 360搜索"甲酰胺 纸尿裤 危害" → 找到中华网等多源报道
→ 心虫推理：甲酰胺毒性有欧盟REACH背书，经皮吸收路径成立
→ 输出：可信度85%，检测方法中等65%，危害真实95%
```

### 与 heartflow_truthfulness skill 的关系

这个模式不是对 truthfulness skill 的替代。truthfulness skill 防止"汇报时编造结果"。
这个模式处理的是"训练数据缺失时的知识补充策略"。

核心区别：
- truthfulness → 先查再说，未核实不开口
- knowledge gap → 心虫不知道≠我（上层LLM）不知道，我可以查了再回答
